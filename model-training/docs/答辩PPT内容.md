## Slide 2 | 技术方案

**标题：技术方案与训练策略**

```
┌──────────────────────────────────────────────────────┐
│                                                      │
│  📊 数据                   🏗️ 模型架构                │
│  ┌─────────────────┐      ┌──────────────────┐      │
│  │ • AI Challenger  │      │ ResNet50 (预训练) │      │
│  │   2018 数据集    │ ───→ │   ↓               │      │
│  │ • ~31,500 训练图 │      │ Dropout(0.2)     │      │
│  │ • 61类→36类映射  │      │   ↓               │      │
│  │ • 合并严重程度   │      │ FC(2048→36)      │      │
│  └─────────────────┘      └──────────────────┘      │
│                                                      │
│  ⚙️ 核心训练策略                                      │
│  ┌────────────────────────────────────────────┐     │
│  │ AdamW + Cosine LR + Warmup   优化器与调度   │     │
│  │ AMP 混合精度                  加速 2×       │     │
│  │ MixUp + Label Smoothing      防过拟合       │     │
│  │ Early Stop (patience=15)     自动收敛       │     │
│  │ 逐层解冻微调                  迁移学习       │     │
│  └────────────────────────────────────────────┘     │
│                                                      │
└──────────────────────────────────────────────────────┘
```

**要点（口述）：**
- 数据集采用 AI Challenger 2018，原始 61 个细粒度类别合并为 36 类（去掉一般/严重区分）
- 使用 ImageNet 预训练 ResNet50 做迁移学习，替换全连接分类头
- 训练策略上，AdamW 优化器 + Cosine 学习率衰减 + Warmup 预热，配合 AMP 混合精度加速
- MixUp 数据增强和 Label Smoothing 防止细粒度分类过拟合

- 基于 2048 维特征向量构建了特征库，实现相似病例检索
- 推理接口已对接后端 FastAPI 服务，形成完整链路
