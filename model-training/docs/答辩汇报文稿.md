各位老师好，我是 XXX，我负责的是项目的**模型训练模块**。下面用大约一分半钟，汇报我的工作。

### 一、问题与方案（约 25 秒）

我们的目标是对 **10 种农作物、36 类病害** 做高精度自动识别。

我选用了 **ResNet50 深度卷积网络** 作为骨干模型，加载 ImageNet 预训练权重做迁移学习。数据来自 AI Challenger 2018 公开数据集，约三万一千张训练图片。原始标注有 61 个细粒度类别，区分了一般和严重程度，但这在实际中容易混淆。我将它们**合并为 36 个分类**，去掉了严重程度维度，让模型更聚焦于病害类型的区分。

### 二、训练策略（约 30 秒）

训练过程中，我搭建了完整的训练管线，核心策略包括六点：

1. **AdamW 优化器** 搭配 **Cosine 学习率衰减** 和 **Warmup 预热**，让训练初期更稳定
2. 开启 **AMP 混合精度训练**，在不损失精度的情况下训练速度提升近一倍
3. 使用 **MixUp 数据增强** 和 **Label Smoothing 标签平滑**，有效缓解 36 类细粒度分类的过拟合
4. 设置 **Early Stop**，验证集 15 轮不提升自动停止，避免无效训练

### 三、效果与交付（约 25 秒）

最终模型在验证集上达到 **90.18% 的 Top-1 准确率**。

工程化方面，我将 PyTorch 模型**导出为 ONNX 格式**，大小从 271MB 压缩到 91MB，支持跨平台 CPU 推理，单张识别在 3 秒内完成。同时构建了**特征库检索系统**，用 2048 维特征向量做余弦相似度匹配，能返回历史上最相似的病例图片作为参考。

最后，我将推理引擎封装为 Python 接口，**输出对接给后端同学的 FastAPI 服务**，形成了从模型训练到线上推理的完整闭环。

以上就是我的工作，谢谢各位老师。

## 💡 临场 tips

- **如果超时**：跳掉「早期停止」和「特征库」细节，保留 90.18% 和 ONNX 部署
- **如果被问 "为什么选 ResNet50 而不是 ViT"**：中小规模数据集上，CNN 的归纳偏置更稳定，ViT 需要更大数据量
- **如果被问 "36 类和 61 类哪个好"**：61 类区分严重程度导致类间混淆，合并后准确率更高、实际更有用
- **如果被问 "你的最大难点"**：细粒度分类的类间差异小（不同病害叶片纹理相似），MixUp + Label Smoothing 是关键解决手段
- **提前准备**：答辩前启动 `model-training/api/api.py`，万一被要求演示可以随时展示 Web UI
