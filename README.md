# 🌾 智慧农业 AI 病虫害识别与种植决策系统

基于深度学习的农作物病害智能识别系统，覆盖**模型训练 → API推理 → RAG知识检索 → Agent种植决策**全链路。

```
│   ├── models/          #   模型定义

│   ├── api/             #   推理服务 + Web UI
│   ├── checkpoints/     #   模型产物
│   └── docs/            #   训练文档

├── frontend/            # 模块3: 前端界面（待开发）

```

## 3.1 作品简介

**作品名称：** 基于深度学习的智慧农业AI病虫害识别与种植决策系统

**作品概述：**

本系统面向农作物病害智能识别场景，基于ResNet50深度卷积神经网络，实现了对苹果、樱桃、玉米、葡萄、柑桔、桃、辣椒、马铃薯、草莓、番茄共10种作物、36类常见病害的高精度自动识别。用户仅需拍摄作物叶片照片，系统即可在2秒内返回病害名称、置信度评分、防治建议及推荐农药，同时支持相似历史病例图片检索，为农户提供可参考的比对依据。

系统采用PyTorch框架完成模型训练，通过ONNX格式导出实现跨平台部署，并提供FastAPI RESTful接口与Web可视化界面，支持云端服务与昇腾端侧推理两种部署模式。整体架构覆盖数据预处理、模型训练与评估、推理服务、知识库查询、相似样例检索五大模块，形成从"拍照到诊断"的完整闭环。

## 3.2 背景及意义

农作物病害是威胁粮食安全和农民收入的重要因素。据联合国粮农组织统计，全球每年因作物病害造成的产量损失高达20%-40%。在我国广大农村地区，植保专家数量有限、分布不均，农户面对叶片异常往往依赖经验判断，误诊率较高，导致用药不当——不仅增加种植成本，还可能造成农药残留超标和环境污染。

当前主要的病害诊断方式有两种：一是农技人员现场勘查，效率低、覆盖面窄，难以满足大面积农田的实时监测需求；二是农户自行上网查阅比对，但缺乏专业知识，容易将相似症状混淆（如番茄早疫病与晚疫病在发病初期极难区分）。

深度学习技术为此提供了理想的解决方案。卷积神经网络能够自动提取叶片病斑的纹理、颜色、形态等高维视觉特征，其识别能力在多项农业AI研究中已达到甚至超过人类专家水平。选择华为昇腾平台的端侧推理方案，是考虑到农田场景中网络覆盖不稳定、数据隐私和实时性三大诉求——模型部署在昇腾开发板上即可离线运行，无需上传图片到云端，既保护农户数据安全，又在无网络条件下提供毫秒级响应。

## 3.3 需求分析

**目标用户群体：**
- 一线种植农户：需要快速、零门槛的病害诊断工具
- 基层农技推广人员：辅助现场勘查，提高诊断准确率
- 植保站/农资店：为客户提供精准用药建议，减少农药滥用

**核心功能需求：**

| 功能 | 描述 |
|------|------|
| 图片上传识别 | 支持拍照或从相册选择，自动预处理并推理 |
| 多分类置信度输出 | 返回Top-5预测结果及置信度百分比 |
| 防治建议生成 | 根据识别结果自动匹配防治方案与推荐农药 |
| 相似病例查看 | 在历史病例库中检索最相似的叶片图片，供农户比对确认 |
| Web可视化界面 | 拖拽上传、结果展示、相似图预览，无需安装App |

**性能指标需求：**
- 识别准确率：≥85%（Top-1 Accuracy）
- 单张推理时间：≤2秒（GPU）/ ≤5秒（CPU端侧）
- 支持离线运行（昇腾开发板端侧部署）
- 模型体积可控（ONNX ≤100MB，便于端侧加载）

## 3.4 采用方案/已采用方案

### 数据来源

使用**AI Challenger 2018 农作物叶子图像数据集**，包含10种作物、61个原始细粒度类别（区分一般/严重程度），训练集约31,500张、验证集约4,500张。为提升模型实用性，将同一病害的"一般"与"严重"标签合并，最终简化为**36个分类类别**。

### 模型构建

基于**ResNet50**骨干网络，加载ImageNet预训练权重进行迁移学习。分类头替换为Dropout(0.2) + Linear(2048→36)结构。训练策略采用：
- **AdamW优化器**（lr=3e-4, weight_decay=1e-4）
- **Cosine学习率衰减 + 2 epoch Warmup**
- **混合精度训练（AMP）**加速
- **Label Smoothing（0.05）**缓解细粒度类别过拟合
- **MixUp数据增强**（前20 epoch），增强泛化能力
- **Early Stop（patience=15）**防止过拟合

### 训练平台

在**华为云ModelArts**平台完成训练，使用Tesla V100 32GB GPU，批量大小64，训练约12个epoch达到最佳验证准确率**90.18%**。

### 部署方案

训练好的PyTorch模型导出为**ONNX**格式（约91MB），支持两种部署模式：
- **云端服务**：FastAPI + Uvicorn提供RESTful API（已实现）
- **端侧推理**：ONNX Runtime或昇腾ACL在昇腾310开发板上本地运行
